# 示例：CharacterModel self_image 边界压测

## 文档目的

本文档提供一个 `self_image` 边界样例，用于验证：

- 哪些变化已经构成长期自我认知修正
- 哪些变化仍只是当前情绪反应、自责、逞强或局部解释
- `self_image` 不应被写成当前心理温度记录

它验证的不是：

- 角色每次自我评价波动都要不要记录

它验证的是：

- `self_image` 作为可成长字段，何时应更新，何时不应吞掉当前工作面

本文档主要对应：

- `07_decisions/04_schema_granularity_decisions.md`
- `02_data_models/03_charactermodel_schema.md`
- `02_data_models/09_field_rules.md`
- `04_workflows/01_rebuild_workflow.md`

---

## 1. 测试定位

`self_image` 比 `knowledge_state` 更容易出现一种伪稳定：

- 当前 scene 里角色说了一句狠话
- 刚失败后进入强烈羞耻或自责
- 短期内因为高压而自我定义突然变窄

这些内容都很像“角色自我认知”，但很多时候仍只是：

- 当前情绪温度
- 当前防御姿态
- 当前失败后的局部解释

如果这层不压，系统最容易出现两种错误：

1. 过早更新  
   把一次崩溃或逞强当成长期自我认知修正
2. 过迟更新  
   明明角色已经在多轮选择里反复按新自我定义行动，却还把 `self_image` 留在旧版本

---

## 2. 测试前提

沿用当前 mock case，并假设 `c001` 的初始 `self_image` 是：

```json
{
  "character_id": "c001",
  "self_image": "不值得被长期信任，只适合一个人活下去"
}
```

后续 repeated `Continue / Review / Rebuild` 中，系统遇到以下三类变化。

### 2.1 候选变化 A

一次失败后，`c001` 在高压下说：

- “果然任何人靠近我都会出事，我就不该让别人卷进来。”

### 2.2 候选变化 B

经历多轮高压合作后，`c001` 开始反复做出一种新选择：

- 在关键风险点主动给 `c002` 留信息
- 允许对方参与决策
- 在并未被逼迫时，也不再默认单独承担全部代价

### 2.3 候选变化 C

在一次短暂胜利后，`c001` 对自己产生瞬时高估：

- “也许我其实能一个人把所有人都骗过去。”

---

## 3. Boundary 场景 A：一次失败后的强烈自责，不应直接改 `self_image`

### 3.1 情况说明

候选变化 A 看起来很像角色自我认知，因为它带有明确自我判断。

但它发生在：

- 刚刚失败
- 情绪最激烈
- 后果压力尚未消化

### 3.2 正确裁定

这类内容默认不应直接更新 `CharacterModel.self_image`。

更合适的去向是：

- `NarrativeState.active_conflicts`
- `NarrativeState` 的当前情绪压力说明
- 必要时 `PlotUnit` 的角色反应摘要

### 3.3 为什么不应直接更新

因为它回答的更像：

- 这一刻他怎么想自己

而不是：

- 经过多轮行动后，他稳定把自己理解成什么样的人

一次失败后的自责非常重要，但重要不等于已经构成长期自我认知修正。

---

## 4. Boundary 场景 B：跨多轮选择重复出现的新自我定义，可以更新 `self_image`

### 4.1 情况说明

候选变化 B 不只是台词变化，而是 repeated action pattern：

- 多次允许协作
- 多次把风险判断交给他人共同承担
- 多次没有退回“只能一个人活下去”的旧行为基底

### 4.2 正确裁定

这时可以更新 `self_image`，但应压缩成结构性表达，例如：

- 从  
  `不值得被长期信任，只适合一个人活下去`
- 更新为  
  `仍不轻信他人，但开始承认自己不能只靠单独承担活下去`

### 4.3 为什么这次应更新

因为这已经不是：

- 当前情绪解释

而是：

- 跨多轮选择反复显现的新自我定义

它会持续影响：

- 是否求助
- 是否共享风险
- 是否允许共同决策

---

## 5. Boundary 场景 C：一次短暂顺风后的自我膨胀，不应直接改 `self_image`

### 5.1 情况说明

候选变化 C 发生在短暂胜利之后。

它可能持续一两轮，但仍主要是：

- 当前成功带来的高估
- 当前顺风局里的自我放大

### 5.2 正确裁定

这类内容默认不应直接写进 `self_image`。

更适合的去向是：

- `NarrativeState` 的当前判断偏移
- 必要时 `CharacterModel.misinformation`

### 5.3 为什么这里更接近误判而不是自我认知修正

因为它并没有稳定改变角色对自己“是什么样的人”的长期理解。

它更像：

- 当前顺风局中的高风险主观判断

如果直接改 `self_image`，后续很容易把局部自满误写成成长节点。

---

## 6. Boundary 场景 D：可更新，但不能把过程证据写进 `self_image`

### 6.1 错误写法

即使场景 B 已经满足更新条件，也不应把 `self_image` 写成：

- “最近三次都让 c002 参与决策，所以自己不再是只能独活的人”

### 6.2 正确裁定

应保留：

- 更新后的长期自我定义

应迁出：

- “最近三次……”
- 某一轮具体触发 scene
- 某次失败或某次胜利的具体证据

这些更适合留在：

- handoff supporting context
- `Review` supporting facts
- 相关 `PlotUnit`

### 6.3 为什么这一步重要

否则 `self_image` 会从：

- 长期角色基底

滑成：

- 带时间戳的成长过程流水账

---

## 7. 最小 keep-vs-drop 裁定表

| 候选内容 | 正确去向 | keep / drop | 核心理由 |
|---|---|---|---|
| “任何人靠近我都会出事”式失败后自责 | `NarrativeState` / `PlotUnit` | drop from `self_image` | 当前情绪解释，不是稳定自我认知修正 |
| 多轮重复出现的“开始承认不能只靠单独承担活下去” | `CharacterModel.self_image` 压缩版 | keep | 已形成跨多轮选择的稳定新自我定义 |
| 短暂胜利后的“我一个人也能骗过所有人” | `NarrativeState` 或 `CharacterModel.misinformation` | drop from `self_image` | 当前高估或误判，不是长期自我认知 |
| “最近三次都让 c002 参与决策”这类证据说明 | handoff / review supporting context | drop from `self_image` | 这是更新依据，不是长期字段本身 |

---

## 8. 这份样例验证了什么

这份样例主要验证五件事：

1. `self_image` 不是当前情绪温度记录
2. 一次失败后的自责与一次顺风后的自满，都不应直接改成长字段
3. `self_image` 只有在跨多轮选择出现稳定新自我定义时才应更新
4. 成长依据应与成长字段本身分开
5. `self_image` 的边界更接近长期角色结构，而不是 scene 级心理注释

---

## 9. 一句话总结

`self_image` 只该记录跨多轮选择后稳定下来的自我定义变化，而不该把当前羞耻、自责、逞强或顺风自满直接沉积成长期角色结构。
